# Scripts Folder Organization

This folder contains all the scripts and tools for the RAG evaluation system, organized into logical subdirectories.

## 📁 Folder Structure

```
scripts/
├── core/               # Core RAG system components
├── evaluation/         # Evaluation datasets and harness
├── generators/         # Question generation tools
├── analysis/           # Analysis and reporting tools
├── results/            # Evaluation results and outputs
├── outputs/            # Generated datasets and questions
└── temp/               # Temporary/test files
```

---

## 📂 Detailed Contents

### `/core` - Core RAG Components
**Purpose**: Essential scripts for running the RAG system

- `rag.py` - Main RAG pipeline (retrieval + generation)
- `ingest.py` - Corpus ingestion into vector database
- `setup_db.py` - Database initialization script
- `demo.py` - Interactive RAG demo

**Usage**:
```bash
# Setup database
python core/setup_db.py

# Ingest corpus
python core/ingest.py

# Run demo
python core/demo.py
```

---

### `/evaluation` - Evaluation Framework
**Purpose**: Test datasets, evaluation harness, and baseline metrics

- `golden_dataset.json` - 75 evaluation questions (50 hand-crafted + 25 synthetic)
- `eval_harness.py` - Full evaluation pipeline with LLM-as-judge
- `baseline_scores.json` - Baseline performance metrics
- `failure_taxonomy.json` - Comprehensive failure analysis

**Usage**:
```bash
# Run full evaluation
python evaluation/eval_harness.py
```

---

### `/generators` - Question Generators
**Purpose**: Tools for generating synthetic test questions

- `synthetic_generator.py` - Generate questions from corpus documents
- `persona_generator.py` - Generate persona-based questions (standard/frustrated/mismatch)

**Usage**:
```bash
# Generate synthetic questions
python generators/synthetic_generator.py

# Generate persona-based questions
python generators/persona_generator.py
```

---

### `/analysis` - Analysis Tools
**Purpose**: Scripts for analyzing results and personas

- `analyze_personas.py` - Compare persona-based evaluation results
- `PERSONA_ANALYSIS.md` - Detailed persona experiment documentation

**Usage**:
```bash
# Analyze persona results
python analysis/analyze_personas.py
```

---

### `/results` - Evaluation Results
**Purpose**: Stored evaluation outputs and metrics

- `eval_results.txt` - Full baseline evaluation output (75 test cases)
- `eval_standard_results.json` - Standard persona results
- `eval_frustrated_results.json` - Frustrated persona results
- `eval_mismatch_results.json` - Mismatch persona results

**These are output files** - regenerated when running evaluations

---

### `/outputs` - Generated Datasets
**Purpose**: Question datasets generated by the system

- `synthetic_questions.json` - Synthetically generated questions (25)
- `persona_standard_questions.json` - Standard persona questions (6)
- `persona_frustrated_questions.json` - Frustrated persona questions (6)
- `persona_mismatch_questions.json` - Mismatch persona questions (6)
- `test_synthetic_output.json` - Test generation output

**These are intermediate outputs** - can be regenerated

---

### `/temp` - Temporary Files
**Purpose**: Test scripts and temporary output files

- `test_synthetic_gen.py` - Test script for synthetic generator
- `eval_personas.py` - Persona evaluation test script
- `*.txt` - Various output logs

**These files are safe to delete** - used for testing/debugging

---

## 🚀 Quick Start

### 1. Setup & Ingest
```bash
cd scripts
python core/setup_db.py
python core/ingest.py
```

### 2. Run Evaluation
```bash
python evaluation/eval_harness.py
```

### 3. Generate Questions
```bash
python generators/synthetic_generator.py
```

### 4. Analyze Results
```bash
python analysis/analyze_personas.py
```

## 🔧 Maintenance

**Regular Tasks**:
- Keep `/core` clean - only essential RAG components
- Archive old results to `/results`
- Clean `/temp` periodically
- Update golden dataset in `/evaluation` as needed

**Adding New Scripts**:
- Core functionality → `/core`
- Evaluation tools → `/evaluation`
- Question generators → `/generators`
- Analysis/reporting → `/analysis`
- Test scripts → `/temp`
